## Setup & Installation

### 1. Prerequisites
- Python 3.11+
- pip
- ServiceNow developer instance (free at developer.servicenow.com)
- Anthropic API key

### 2. Clone and install

```bash
git clone <repo-url>
cd cmdb_ai_project
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
SN_INSTANCE=https://your-instance.service-now.com
SN_USERNAME=admin
SN_PASSWORD=your_password
ANTHROPIC_API_KEY=sk-ant-...
FLASK_SECRET=your-secret-key
```

Or export them directly:

```bash
export SN_INSTANCE="https://dev12345.service-now.com"
export SN_USERNAME="admin"
export SN_PASSWORD="YourPassword123"
export ANTHROPIC_API_KEY="sk-ant-api03-..."
```

### 4. Train the ML model

```bash
python train_model.py
```

Expected output:
```
[INFO] Loading dataset from: datasets/cmdb_training_data.csv
[INFO] Loaded 100 records
[INFO] Accuracy: 0.9500 (95.00%)
[INFO] Model saved → models/cmdb_classifier.pkl
[INFO] ✅  Model training pipeline complete.
```

### 5. Run the Flask application

```bash
python app.py
```

Open browser: **http://localhost:5000**

## Application Workflow

```
Step 1: Upload
  → Navigate to Upload page
  → Drag & drop or select CSV/Excel file
  → File must contain: name, os, manufacturer, description, environment

Step 2: Predict
  → Click "Run Prediction"
  → RandomForest classifies each CI into server/database/network/cloud/application/storage
  → Confidence score and CMDB table displayed per record

Step 3: Approve
  → Check boxes next to records to approve
  → Click "Push to ServiceNow"

Step 4: Results
  → Dashboard shows success/error counts
  → Charts show CI class and environment distribution
  → Insert log shows sys_id for each inserted record

Step 5: Chat (optional)
  → Ask natural language questions
  → AI queries CMDB and explains results
```

## ServiceNow REST API Reference

### Insert CI record

```
POST https://<instance>.service-now.com/api/now/table/cmdb_ci_server

Headers:
  Content-Type: application/json
  Accept: application/json
  Authorization: Basic <base64(user:pass)>

Body:
{
  "name": "web-server-01",
  "ip_address": "10.0.1.10",
  "os": "Ubuntu 22.04 LTS",
  "manufacturer": "Dell",
  "description": "Apache web server",
  "environment": "production"
}

Response:
{
  "result": {
    "sys_id": "abc123...",
    "name": "web-server-01",
    "sys_class_name": "cmdb_ci_server"
  }
}
```

### Query CMDB

```
GET https://<instance>.service-now.com/api/now/table/cmdb_ci_server
    ?sysparm_query=osLIKELinux
    &sysparm_fields=name,ip_address,os,environment
    &sysparm_limit=50
    &sysparm_display_value=true
```

## ML Model Details

### Algorithm: RandomForestClassifier
- **Estimators**: 200 decision trees
- **Class weighting**: balanced (handles class imbalance)
- **Features**: os, manufacturer, description, environment
- **Encoding**: LabelEncoder per feature
- **Split**: 80% train / 20% test

### Expected Performance
- Accuracy: ~90–98% (depending on dataset variety)
- Best for: Clearly described infrastructure with standard OS/manufacturer values
- Handles unseen labels gracefully (maps to index 0, still predicts)


```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:8000", "app:app"]
```

### Environment security checklist
- [ ] Store credentials in environment variables, not code
- [ ] Use HTTPS in production (TLS certificate)
- [ ] Rotate ServiceNow credentials regularly
- [ ] Restrict ServiceNow API user to CMDB write permissions only
- [ ] Set Flask `SECRET_KEY` to a cryptographically random value

## CSDM Best Practices Followed

1. **OOB tables only** — Uses out-of-box CMDB tables, no custom tables
2. **Upgrade-safe** — No customisations to base system fields
3. **Minimal payload** — Only known OOB fields are sent per table
4. **Retry handling** — Exponential backoff on 5xx/429 responses
5. **sys_id capture** — All inserted records tracked by sys_id
6. **Environment tagging** — production/development/staging populated correctly
7. **Field mapping** — Table-specific field maps ensure correct schema population

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `Model not found` error | Run `python train_model.py` first |
| ServiceNow 401 error | Check `SN_USERNAME` / `SN_PASSWORD` env vars |
| ServiceNow 403 error | Ensure API user has `cmdb_write` role |
| File upload error | Check file has required columns (name, os, manufacturer, description, environment) |
| Chatbot no response | Check `ANTHROPIC_API_KEY` env var is set |
| Low prediction confidence | Ensure description column has meaningful text |
